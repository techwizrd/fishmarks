# Security Policy

## Supported versions

Security fixes are provided for the latest release on `master`.

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Instead:

1. Use GitHub's private vulnerability reporting for this repository, or
2. Contact the maintainers privately through repository administrators.

Include as much detail as possible:

- A clear description of the issue
- Reproduction steps or proof of concept
- Potential impact
- Suggested mitigation (if known)

We will acknowledge reports promptly and coordinate a fix and disclosure timeline.
